Materializing an RDF view from an RML mapping

This command-line tool materializes an RDF dataset from an RML mapping. It reads an RML description (Triples Maps), evaluates the mapping against its referenced JSON sources and outputs the resulting RDF dataset.

You can use the program as follows, assuming the file MyMapping.ttl contains your RML mapping. The tool reads this mapping, materializes all rml:TriplesMap definitions and writes the resulting RDF dataset to standard output or a file. It also assumes that you have added the HeFQUIN programs to your PATH (otherwise, enter the directory in which you have unpacked the HeFQUIN release and replace hefquin-rmlmat by bin/hefquin-rmlmat). 

hefquin-rmlmat --mapping=MyMapping.ttl

Further arguments can be passed to the program, which are described in detail below. You can also have the list of supported arguments printed by executing the program with the argument --help. 

hefquin-rmlmat --help

Arguments

--mapping=<rdf-file> (mandatory argument)

Path to an RDF file containing the RML mapping. The file must include rml:TriplesMap definitions. All Triples Maps found in the file will be processed during materialization. Supported RDF serializations include Turtle, RDF/XML, JSON-LD and N-Triples.

--outputToFile=<file-name>

Optional output destination file. If omitted, results are written to standard output. If the file already exists, output is appended rather than overwritten.

--baseIRI=<iri>

Optional base IRI used during mapping expansion and IRI resolution. If not provided, an internal default base IRI is used.

--time

Enables execution timing. When active, the tool prints the total time spent in the core materialization phase (mapping evaluation + RDF generation).

--help

If this argument is given, the program prints a list of all supported arguments and terminates (i.e., all other given arguments are ignored).
