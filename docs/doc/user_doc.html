User documentation

HeFQUIN can be deployed as a web service, run directly from the command line, or run in Docker container. If you are looking to add new functionality to the engine see Getting started as a contributor.

Run HeFQUIN using the CLI

HeFQUIN includes various command-line utilities that allow you to execute single queries, inspect execution plans, start HeFQUIN as an embedded web service, and more. Begin by downloading the latest HeFQUIN release (see Releases).

You can start using the CLI tools directly by simply referencing the target scripts from the command line. However, it's recommended that you define the environment variable HEFQUIN_HOME and add the scripts to the PATH variable. You can set this up as follows:

On Linux / Mac:


export HEFQUIN_HOME="the dist/ directory where you downloaded HeFQUIN"
export PATH=$PATH:$HEFQUIN_HOME/bin

On Windows:


SET HEFQUIN_HOME ="the dist/ directory where you downloaded HeFQUIN"
SET PATH=%PATH%;%HEFQUIN_HOME%\bat

Commands:

  • hefquin - Run a single query
  • hefquin-server - Start the engine as an embedded web service (defaults to http://localhost:8080/)
  • hefquin-pg - Run a SPARQL query over a PG store
  • hefquin-pg - Materialize an RDF view of a PG store

To see usage instructions and available arguments for each command, execute the command followed by --help.

Deploy as a Web service

HeFQUIN can be deployed as a web service in servlet container, such as Jetty, Tomcat, or GlassFish. Start by downloading the latest HeFQUIN release (see Releases). Deploy hefquin-service-x-y-z.war in to your servlet container. To configure the HeFQUIN engine, customized versions of the files DefaultEngineConf.ttl and DefaultFedConf.ttl to the root of the deployed servlet.

Note: The servlet will need to be restarted for any changes in the engine configuration to take effect.

Run using Docker

HeFQUIN can also be run as Docker container. Pull the latest release using: 

docker pull ghcr.io/LiUSemWeb/hefquin:latest

Then run the engine and optionally mount custom descriptions of the engine configuration and federation description: 


docker run \
    -p 8080:8080 \
    -v MyEngineConf.ttl:/usr/local/tomcat/webapps/ROOT/DefaultEngineConf.ttl \
    -v MyFedConf.ttl:/usr/local/tomcat/webapps/ROOT/DefaultFedConf.ttl \
    hefquin:latest
